# 성능/UX 리뷰 보고서 - 에이전트 B

범위: 프론트엔드 UX, 체감 성능, 렌더링, 사용자 상호작용.

## 핵심 발견

1. 메인 목록과 마이페이지 이미지에 lazy loading, async decoding, 안정적인 이미지 크기가 없다. 페이지당 여러 이미지가 즉시 로드되어 모바일 첫 렌더링과 데이터 사용량에 불리하다.

2. 마이페이지는 보이지 않는 탭의 카드와 이미지까지 한 번에 SSR한다. 기본 탭만 렌더링하거나 탭 클릭 시 API로 가져오는 구조가 좋다.

3. 검색/페이지네이션 링크가 EJS에서 수동 query string으로 조립된다. 검색어에 `&`, `+`, 공백, 한국어가 섞이면 필터 보존이 깨질 수 있다.

4. 검색 제출은 전체 페이지 reload이며 로딩 상태가 없다. 느린 쿼리에서 사용자는 진행 중인지 알기 어렵고 중복 제출할 수 있다.

5. 모바일 카드가 작은 폭에서도 정보와 액션 4개를 동시에 보여준다. 오탭 위험이 있고 비교/선택 경험이 답답해질 수 있다.

6. 토글/리뷰/사용자 추가 실패 피드백이 `alert` 중심이라 모바일 흐름을 끊는다. toast/snackbar와 버튼 내부 pending 상태가 더 적합하다.

7. 정적 파일 캐시 정책과 Google Fonts 로딩 전략이 명시적으로 최적화되어 있지 않다.

## 권장 순서

1. 이미지 lazy/async/size 지정과 레이아웃 안정화를 먼저 적용한다.
2. 페이지네이션 URL 생성을 서버 helper로 통일한다.
3. 검색 제출 pending 상태를 추가한다.
4. 마이페이지 탭별 lazy load 또는 탭별 limit를 도입한다.
5. 모바일 카드 레이아웃과 toast UX를 개선한다.
